Recursividade

🪆

Pense nas bonecas russas (matrioscas): cada boneca contém uma menor dentro de si, até chegar à menor de todas, que não abre mais. Essa menor é o caso-base: o ponto de parada. A recursividade é exatamente isso: uma função que resolve um problema chamando a si mesma com uma versão menor do mesmo problema, até atingir o caso-base. Em telecomunicações, antenas fractais repetem a mesma forma geométrica em escalas cada vez menores, ganhando largura de banda sem aumentar o tamanho físico. O mesmo padrão de "resolver dividindo em versões menores de si mesmo" aparece em algoritmos de ordenação, compressão e muito mais.

Uma função que chama a si mesma

Em Python, uma função pode chamar a si mesma. Isso é recursividade. Para não chamar indefinidamente, ela precisa de duas partes:

  • Caso-base: condição de parada, quando a função retorna sem chamar a si mesma.
  • Passo recursivo: a função chama a si mesma com uma entrada menor, caminhando em direção ao caso-base.

O exemplo clássico é a contagem regressiva. Compare a versão recursiva com um laço mental:

Python 
def contagem_regressiva(n):
    if n == 0:          # caso-base: para aqui
        print("Fim")
        return
    print(n)
    contagem_regressiva(n - 1)  # passo recursivo: n menor

contagem_regressiva(5)

Cada chamada imprime o número atual e depois delega o resto para uma versão com n - 1. Quando n chega a 0, o caso-base encerra a cadeia.

Fatorial recursivo

O fatorial (n! = n × (n−1) × … × 1) tem uma definição naturalmente recursiva: fatorial(n) = n × fatorial(n−1), com fatorial(1) = 1 como caso-base.

Python 
def fatorial(n):
    if n <= 1:            # caso-base
        return 1
    return n * fatorial(n - 1)  # passo recursivo

print(fatorial(5))   # 120
print(fatorial(10))  # 3628800

Compare com a versão em laço: o resultado é idêntico, mas a versão recursiva espelha diretamente a definição matemática, o que a torna mais fácil de verificar e de provar correta:

Python 
# Versão com laço (para comparar)
def fatorial_laco(n):
    resultado = 1
    for k in range(2, n + 1):
        resultado *= k
    return resultado

print(fatorial_laco(5))   # 120: mesmo resultado
Sem caso-base, a recursão nunca termina: assim como um while sem condição de saída trava o programa em laço infinito, uma função recursiva sem caso-base se chama para sempre até esgotar a memória da pilha (stack overflow). Garanta sempre que o passo recursivo diminua o problema em direção ao caso-base.

Na engenharia

Recursividade aparece onde problemas têm estrutura autossimilar. Veja o padrão nas três áreas.

⚙️ Mecânica

Fatorial recursivo, base para combinações em análise de tensões e probabilidades:

Python 
def fatorial(n):
    return 1 if n <= 1 else n * fatorial(n - 1)

print(fatorial(6))   # 720: C(10,6) usa 10!/(6!×4!)
⚡ Elétrica

Soma de série geométrica recursiva, modela atenuação por estágios em filtros:

Python 
def soma_serie(n):
    return 0 if n == 0 else 1 / (2**n) + soma_serie(n - 1)

print(f"{soma_serie(10):.5f}")  # ≈ 0.99902 → converge para 1
📡 Telecomunicações

Fibonacci recursivo, modela crescimento em redes e sistemas de ramificação:

Python 
def fib(n):
    return n if n < 2 else fib(n - 1) + fib(n - 2)

print(fib(10))  # 55

Caso real de engenharia: atenuação acumulada por estágios

Muitos sistemas se definem em termos de si mesmos. Aqui cada estágio de um filtro reduz o sinal à metade; a função recursiva calcula o sinal após N estágios.

Python 
def sinal(estagios, valor=1.0):
    if estagios == 0:          # caso-base: nenhum estagio
        return valor
    return sinal(estagios - 1, valor / 2)   # passo recursivo

for n in range(0, 6):
    print(f"{n} estagios -> sinal = {sinal(n):.4f}")
Toda recursão precisa de um caso-base (a parada) e de um passo que se aproxima dele, como nos N estágios chegando a zero.

15. (geral) A recursão ingênua de Fibonacci faz muitas chamadas repetidas. Meça o custo: conte quantas chamadas fib(n) faz para cada n de 1 a 15 usando um contador em lista mutável. Imprima e depois plote o crescimento do custo.

Mostrar solução
Sem gráfico (texto)
Python 
def fib_conta(n, contador):
    contador[0] += 1
    if n < 2:
        return n
    return fib_conta(n - 1, contador) + fib_conta(n - 2, contador)

for n in range(1, 16):
    c = [0]
    fib_conta(n, c)
    print(f"fib({n}) custou {c[0]} chamadas")
Com gráfico (matplotlib)
Python 
import matplotlib.pyplot as plt

def fib_conta(n, contador):
    contador[0] += 1
    if n < 2:
        return n
    return fib_conta(n - 1, contador) + fib_conta(n - 2, contador)

ns = list(range(1, 16))
custos = []
for n in ns:
    c = [0]
    fib_conta(n, c)
    custos.append(c[0])
plt.plot(ns, custos, marker="o")
plt.title("Custo da recursão ingênua de Fibonacci")
plt.xlabel("n")
plt.ylabel("nº de chamadas")
plt.grid(True)
plt.show()

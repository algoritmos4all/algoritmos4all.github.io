Um algoritmo é como uma receita invisível que roda dentro das coisas. O drone que se equilibra no vento, o painel solar que extrai o máximo de energia, o celular que conversa com uma antena a quilômetros de distância: por trás de cada um existe um algoritmo decidindo, corrigindo e otimizando milhares de vezes por segundo.

Estes exemplos são versões simplificadas, feitas para caber em poucas linhas e serem entendidas. Os sistemas reais usam as mesmas ideias em escala muito maior, mas a semente é exatamente o que você vê aqui.

⚙️ Mecânica: peças mais leves projetadas por busca

Na manufatura aditiva (impressão 3D de metais) e no design generativo, o computador não espera o engenheiro desenhar a peça: ele testa milhares de variações e fica com a que pesa menos sem quebrar. Suportes de aeronaves redesenhados assim chegam a ficar até 45% mais leves, o que economiza combustível em cada voo.

O coração disso é simples: um laço que percorre as possibilidades, uma condicional que descarta as que não aguentam a carga e um acumulador que guarda a melhor. Procure a viga mais leve que ainda suporta o esforço:

Python 
# Achar a viga retangular mais LEVE que ainda aguenta a carga.
carga = 5000.0          # força aplicada, em newtons
comprimento = 1.0       # metros
tensao_max = 200e6      # limite do material, em pascals
densidade = 2700.0      # alumínio, em kg/m3

melhor = None
# Busca por força bruta: testa várias larguras e alturas
for largura_mm in range(10, 61, 2):
    for altura_mm in range(10, 61, 2):
        b = largura_mm / 1000
        h = altura_mm / 1000
        momento = carga * comprimento / 4      # flexão no meio da viga
        modulo = b * h**2 / 6                   # resistência da seção
        tensao = momento / modulo
        if tensao <= tensao_max:               # só aceita se aguentar
            massa = b * h * comprimento * densidade
            if melhor is None or massa < melhor[0]:
                melhor = (massa, largura_mm, altura_mm, tensao)

massa, b_mm, h_mm, tensao = melhor
print(f"Viga ótima: {b_mm} mm de largura x {h_mm} mm de altura")
print(f"Massa: {massa:.3f} kg")
print(f"Tensão: {tensao/1e6:.0f} MPa (limite de 200 MPa)")
Repare: o programa não "sabe" a resposta de antemão. Ele experimenta e compara. É assim que um algoritmo de otimização encontra soluções que o intuito humano dificilmente acharia sozinho.

🛸 Drones e carros autônomos: confiar em sensores ruidosos

Todo sensor mente um pouquinho: GPS treme, acelerômetros vibram, radares têm interferência. Drones e veículos autônomos sobrevivem porque rodam algoritmos de fusão de sensores (o mais famoso é o filtro de Kalman) que combinam muitas leituras imperfeitas em uma estimativa estável. Sem isso, um drone oscilaria sem parar e cairia.

A versão mais simples dessa ideia é um filtro que mistura cada nova leitura com a estimativa anterior. Rode e veja o sinal ruidoso (cinza) virar uma curva suave (vermelha):

Python 
import random
import matplotlib.pyplot as plt

# A distância "real" sobe devagar, mas o sensor vem cheio de ruído.
medido = []
for n in range(60):
    valor_real = 10 + 3 * (n / 60)
    ruido = random.uniform(-1.5, 1.5)
    medido.append(valor_real + ruido)

# Filtro: cada estimativa é uma mistura da nova leitura com a anterior.
alfa = 0.2
estimativa = medido[0]
filtrado = []
for leitura in medido:
    estimativa = alfa * leitura + (1 - alfa) * estimativa
    filtrado.append(estimativa)

plt.plot(medido, ".", color="gray", label="sensor (ruidoso)")
plt.plot(filtrado, color="red", linewidth=2, label="estimativa filtrada")
plt.title("Fusão de sensores em um drone")
plt.xlabel("amostra")
plt.ylabel("distância (m)")
plt.legend()
plt.grid(True)
plt.show()

O parâmetro alfa regula a confiança: perto de 1, o filtro acredita muito no sensor (responde rápido, mas treme); perto de 0, ele confia na história (suave, mas lento). Experimente trocar alfa para 0.5 e rodar de novo.

📡 Telecomunicações: corrigir erros sem pedir para repetir

Quando seu celular recebe dados de uma antena 5G, ou quando a sonda Voyager envia uma foto de bilhões de quilômetros de distância, alguns bits chegam corrompidos pelo ruído. Pedir para reenviar nem sempre é possível. A solução são os códigos corretores de erro: enviam-se bits extras, calculados de forma esperta, que permitem ao receptor detectar e consertar o erro sozinho. O 5G usa códigos LDPC; a ideia mais simples de todas é a repetição com voto da maioria.

Python 
import random

def codificar(bit):
    return [bit, bit, bit]          # repete cada bit 3 vezes

def canal_ruidoso(bits):
    saida = []
    for b in bits:
        if random.random() < 0.2:   # 20% de chance de inverter o bit
            saida.append(1 - b)
        else:
            saida.append(b)
    return saida

def decodificar(bits):
    return 1 if sum(bits) >= 2 else 0   # voto da maioria

mensagem = [1, 0, 1, 1, 0]
recuperada = []
for bit in mensagem:
    enviado = codificar(bit)
    chegou = canal_ruidoso(enviado)
    recuperada.append(decodificar(chegou))

print("Mensagem original: ", mensagem)
print("Mensagem recebida: ", recuperada)
print("Recuperada sem erro?", mensagem == recuperada)
Rode várias vezes. Mesmo com 20% dos bits corrompidos no caminho, a mensagem quase sempre chega inteira. É essa mágica que permite ligações claras, QR codes arranhados que ainda funcionam e o streaming não travar a cada falha.

☀️ Energia elétrica: extrair o máximo do Sol

Um painel solar entrega potência diferente conforme a tensão de operação, e existe um único ponto em que a potência é máxima, que ainda muda com o clima. Por isso todo inversor solar moderno roda, a cada instante, um algoritmo de MPPT (rastreamento do ponto de máxima potência). A técnica clássica é a "subida de encosta": dar passinhos na direção que aumenta a potência.

Python 
import matplotlib.pyplot as plt

# Modelo simples da curva de potência do painel (sobe e depois cai).
def potencia(tensao):
    if tensao < 0 or tensao > 40:
        return 0
    return tensao * (40 - tensao) * 0.5     # watts

# Subida de encosta: avança enquanto a potência crescer.
tensao = 5.0
passo = 1.0
for _ in range(40):
    if potencia(tensao + passo) > potencia(tensao):
        tensao = tensao + passo
    else:
        tensao = tensao - passo
        passo = passo / 2                   # refina a busca

print(f"Ponto de máxima potência: {tensao:.1f} V")
print(f"Potência extraída: {potencia(tensao):.0f} W")

vs = [v / 2 for v in range(0, 81)]
ps = [potencia(v) for v in vs]
plt.plot(vs, ps, label="curva do painel")
plt.plot(tensao, potencia(tensao), "ro", markersize=10, label="ponto encontrado")
plt.title("MPPT: busca do ponto de máxima potência")
plt.xlabel("tensão (V)")
plt.ylabel("potência (W)")
plt.legend()
plt.grid(True)
plt.show()

🗺️ Logística e internet: o caminho mais curto

Quando o GPS calcula sua rota, quando um app de entrega decide a ordem das paradas ou quando a internet escolhe por onde mandar um pacote, o que roda por baixo é um algoritmo de menor caminho (como o de Dijkstra). Ele transforma o mapa em um grafo de pontos e custos e descobre o trajeto mais barato. Veja em ação com um mapinha de uma cidade:

Python 
# Mapa: cada rua liga dois pontos e tem um custo (tempo em minutos).
mapa = {
    "Casa":    {"Praca": 4, "Mercado": 2},
    "Mercado": {"Praca": 1, "Escola": 7},
    "Praca":   {"Escola": 3},
    "Escola":  {},
}

def menor_caminho(mapa, inicio, destino):
    distancia = {ponto: float("inf") for ponto in mapa}
    distancia[inicio] = 0
    anterior = {}
    falta = set(mapa)

    while falta:
        atual = min(falta, key=lambda p: distancia[p])  # o mais próximo ainda não visitado
        falta.remove(atual)
        for vizinho, custo in mapa[atual].items():
            novo = distancia[atual] + custo
            if novo < distancia[vizinho]:
                distancia[vizinho] = novo
                anterior[vizinho] = atual

    caminho = [destino]                                 # reconstrói de trás para frente
    while caminho[-1] != inicio:
        caminho.append(anterior[caminho[-1]])
    caminho.reverse()
    return caminho, distancia[destino]

rota, tempo = menor_caminho(mapa, "Casa", "Escola")
print("Melhor rota:", " -> ".join(rota))
print("Tempo total:", tempo, "min")

O caminho direto Casa → Mercado → Escola levaria 9 minutos, mas o algoritmo descobre que passar pela Praça é mais rápido. É a mesma lógica, em escala planetária, que mantém a internet e a logística mundial funcionando.

⚡ Elétrica

O 5G usa códigos LDPC para corrigir erros e formatação de feixe para mirar a antena; a FFT separa os sinais; e algoritmos de menor caminho roteiam cada pacote da internet. A Voyager ainda fala com a Terra graças a códigos corretores.

O fio condutor: em todos esses casos, a inteligência não está no hardware sozinho, está no algoritmo que decide o que fazer. E algoritmo é exatamente o que você aprendeu a escrever neste curso.

🟢 Aquecimento

1. No exemplo do código corretor de erros, troque a chance de erro de 0.2 para 0.45 e rode algumas vezes. A mensagem ainda costuma chegar inteira? Por quê?

Mostrar solução

Com 45% de chance de erro, frequentemente dois dos três bits são invertidos, e aí o voto da maioria erra. Um código de repetição de 3 só corrige bem quando os erros são raros: por isso o 5G usa códigos bem mais sofisticados, capazes de corrigir muito mais com poucos bits extras.

🟢 Aquecimento

2. No filtro do drone, mude alfa para 0.05 e depois para 0.8. Descreva o que muda na curva vermelha.

Mostrar solução

Com alfa = 0.05 a curva fica muito suave, mas demora a acompanhar a subida real (resposta lenta). Com alfa = 0.8 ela acompanha rápido, porém volta a tremer junto com o ruído. O bom valor é um equilíbrio entre suavidade e rapidez.

🔴 Desafio de engenharia

5. (⚙️ Mecânica) Um controlador (como o de uma impressora 3D ou de um forno) precisa levar uma temperatura até um valor desejado sem ultrapassar muito. Simule um controle proporcional simples: a cada passo, corrija a temperatura por uma fração do erro até o desejado. Plote a curva chegando ao alvo.

Mostrar solução
Python 
import matplotlib.pyplot as plt

alvo = 200.0          # temperatura desejada (graus C)
temp = 20.0           # começa fria
ganho = 0.15          # quão forte é a correção

historico = []
for passo in range(60):
    historico.append(temp)
    erro = alvo - temp
    temp = temp + ganho * erro   # corrige proporcional ao erro

plt.plot(historico, marker=".", label="temperatura")
plt.axhline(alvo, color="red", linestyle="--", label="alvo")
plt.title("Controle proporcional de temperatura")
plt.xlabel("passo")
plt.ylabel("temperatura (graus C)")
plt.legend()
plt.grid(True)
plt.show()

A temperatura sobe rápido no início (erro grande) e desacelera ao chegar perto do alvo (erro pequeno). É a base do controle PID, presente em quase toda máquina que precisa manter uma grandeza estável.

🔴 Desafio de engenharia

6. (⚡ Elétrica / 📡 Telecom) Some dois sinais senoidais de frequências diferentes (por exemplo 2 Hz e 5 Hz) e plote o resultado. Esse é o tipo de sinal misturado que a transformada de Fourier sabe separar de volta em suas frequências.

Mostrar solução
Python 
import math
import matplotlib.pyplot as plt

ts = []
sinal = []
t = 0.0
while t <= 1.0:
    onda1 = math.sin(2 * math.pi * 2 * t)   # 2 Hz
    onda2 = 0.5 * math.sin(2 * math.pi * 5 * t)   # 5 Hz, mais fraca
    ts.append(t)
    sinal.append(onda1 + onda2)
    t = t + 0.005

plt.plot(ts, sinal)
plt.title("Soma de dois sinais senoidais")
plt.xlabel("t (s)")
plt.ylabel("amplitude")
plt.grid(True)
plt.show()

O resultado é uma onda complicada que esconde duas frequências. Separar essa mistura de volta é exatamente o que a FFT faz, e é por isso que ela é um dos algoritmos mais usados do mundo, do áudio digital às redes 5G.

O que é um algoritmo

⚙️

Pense numa linha de montagem industrial: cada operador executa sua etapa na ordem exata definida pelo engenheiro: apertar o parafuso antes de instalar a tampa, nunca ao contrário. Ou pense na rotina de partida de um motor: aquecer, verificar pressão de óleo, acionar o acionamento, a ordem importa e cada passo precisa ser preciso. Um algoritmo é exatamente essa sequência de instruções claras e ordenadas que, seguida à risca, resolve um problema.

Antes de escrever qualquer código, vale entender a ideia mais importante de toda a programação: o algoritmo.

Um algoritmo é simplesmente uma sequência de passos para resolver um problema ou realizar uma tarefa. Você já segue algoritmos o dia todo, mesmo sem perceber. Uma receita de bolo é um algoritmo. As instruções para chegar à casa de um amigo são um algoritmo.

Em uma frase: programar é descrever, com muita clareza e na ordem certa, os passos que o computador deve seguir.

Um exemplo do dia a dia

Pense no algoritmo para "fazer um sanduíche":

  1. Pegar duas fatias de pão.
  2. Passar manteiga em uma fatia.
  3. Colocar o queijo sobre a manteiga.
  4. Cobrir com a outra fatia.

Repare em duas coisas: a ordem importa (não dá para cobrir antes de colocar o queijo) e os passos precisam ser precisos. O computador é muito obediente, mas nada esperto: ele faz exatamente o que você manda, na ordem em que você manda.

O primeiro algoritmo em Python

Em Python, o comando print() mostra uma mensagem na tela. Este é o tradicional "primeiro programa" de quem aprende a programar. Clique em Executar para vê-lo funcionar:

Python 
print("Olá, mundo!")
print("Estou aprendendo a programar.")

Cada linha é um passo. O computador executa de cima para baixo: primeiro mostra a primeira mensagem, depois a segunda. Esse é o nosso algoritmo em ação.

Por que a ordem importa

Experimente trocar a ordem das linhas abaixo (você pode editar o código antes de executar) e veja como a saída muda:

Python: edite à vontade 
print("Bom dia!")
print("Como vai você?")
print("Até logo!")
Dica: o código aqui é editável. Errar não tem problema nenhum, pelo contrário, é assim que se aprende. Mude, execute, observe.

Na engenharia

Algoritmos sequenciais aparecem em toda a engenharia. Veja três exemplos: cada um é uma sequência de passos em ordem obrigatória.

⚙️ Mecânica

Sequência de partida de uma máquina industrial (cada passo na ordem certa garante segurança):

Python 
print("1. Travar a peça")
print("2. Verificar pressão de óleo")
print("3. Ligar o motor")
print("4. Iniciar o corte")
⚡ Elétrica

Procedimento seguro para energizar um circuito (a ordem evita curtos e choques):

Python 
print("1. Verificar isolação dos fios")
print("2. Fechar o disjuntor de proteção")
print("3. Ligar a fonte de alimentação")
print("4. Confirmar tensão com multímetro")
📡 Telecomunicações

Passos para transmitir uma mensagem com confirmação de entrega:

Python 
print("1. Codificar a mensagem")
print("2. Transmitir pelo canal")
print("3. Aguardar confirmação (ACK)")
print("4. Registrar entrega bem-sucedida")

Caso real de engenharia: rotina de partida de uma bomba

A ordem dos passos não é decoração: inverter a sequência pode danificar o equipamento. Este algoritmo executa a partida na ordem segura e só confirma a operação depois que cada etapa foi cumprida.

Python 
# Rotina de partida (cada passo depende do anterior)
print("1. Verificando nivel do reservatorio... OK")
print("2. Abrindo valvula de succao... OK")
print("3. Conferindo pressao do oleo... OK")
print("4. Energizando o motor... OK")
print("5. Confirmando vazao na saida... OK")
print("Bomba operando com seguranca.")
A força de um algoritmo sequencial está na ordem. Trocar dois passos de lugar muda o resultado, exatamente como numa receita ou num procedimento de campo.

🔴 Desafio de engenharia

10. (⚙️ Mecânica) Descreva o algoritmo de inspeção de uma peça mecânica antes da montagem. Liste pelo menos 4 passos em ordem, como um engenheiro de qualidade faria.

Mostrar uma solução possível
  1. Verificar se as dimensões da peça estão dentro da tolerância.
  2. Inspecionar visualmente a superfície em busca de trincas ou rebarbas.
  3. Testar o encaixe da peça no ponto de montagem.
  4. Registrar o resultado da inspeção na ficha de qualidade.
  5. Liberar a peça para montagem (ou rejeitar se houver falha).

Este é um algoritmo de controle de qualidade: cada passo em ordem garante que nenhuma peça defeituosa passe para a linha de montagem.

🔴 Desafio de engenharia

11. (⚡ Elétrica) Descreva o algoritmo de procedimento seguro para medir tensão com um multímetro. Liste os passos em ordem.

Mostrar uma solução possível
  1. Verificar se o multímetro está em bom estado (bateria, ponteiras).
  2. Selecionar a faixa de tensão adequada ao circuito a medir.
  3. Conectar a ponteira preta no terminal COM e a vermelha no terminal V.
  4. Encostar as ponteiras nos pontos do circuito a medir.
  5. Ler o valor no display e anotar o resultado.
  6. Desconectar as ponteiras e desligar o multímetro.

A ordem importa: conectar na faixa errada antes de medir pode danificar o equipamento. Algoritmos de segurança elétrica são seguidos à risca.

🔴 Desafio de engenharia

12. (📡 Telecomunicações) Descreva em linguagem natural o algoritmo para transmitir uma mensagem com confirmação de recebimento, incluindo o que fazer se a confirmação não chegar. (Não é necessário escrever código: descreva o raciocínio com passos numerados.)

Mostrar uma solução possível
  1. Codificar a mensagem no formato adequado ao canal.
  2. Transmitir a mensagem pelo canal de comunicação.
  3. Aguardar a confirmação de recebimento (ACK) por um tempo limite.
  4. Se a confirmação chegar: registrar a transmissão como bem-sucedida e encerrar.
  5. Se o tempo limite esgotar sem confirmação: reenviar a mensagem (voltar ao passo 2).
  6. Após um número máximo de tentativas sem sucesso: registrar falha e alertar o operador.

Repare que o passo 5 faz o algoritmo "voltar": isso é a ideia de repetição, que você vai aprender a programar nos próximos módulos.

Prévia: aonde isso leva

Você ainda não precisa entender este código: execute e veja um algoritmo desenhando um gráfico, algo que você vai aprender a fazer mais adiante.

Python 
import matplotlib.pyplot as plt
plt.plot([1, 2, 3, 4, 5], [1, 4, 9, 16, 25], marker="o")
plt.title("Um algoritmo desenhando um gráfico")
plt.xlabel("número")
plt.ylabel("seu quadrado")
plt.grid(True)
plt.show()
